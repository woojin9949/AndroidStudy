Activity
- 화면을 구성하는 가장 기본이 되는 요소
- 자동으로 Activity와 매칭되는 xml을 생성해줌 + manifest에도 등록이 된다
- 화면을 그리는 기능 + 화면에서 발생할 수 있는 일들을 처리할 수 있는 기능 내장

- 특수한 Activity
    - 런처 activity, main activity
        - 앱이 실행될때 최초로 실행되는 activity

- LifeCycle(생명주기)
    - 안드로이드 OS가 activity를 직접적으로 종료시키는 일은 없다
        - BUT 프로세스는 종료 가능

- onCreate
    - 필수적으로 구현을 해야한다
    - 생명 주기 중에서 단 한번만 발생한다
        - 단 한번만 하면 되는 일들을 여기서 구현한다
        - 화면 그리기, 데이터 준비
- onStart
    - activity를 포그라운드로 보내 상호작용을 할 수 있도록 준비(foreground)
        - 포그라운드 : 사용자 눈에 보일때 또는 사용중일때
    - 매우 빠르게 완료가 된다
- onResume (실질적 휴대폰 앱 사용중일때)
    - 앱이 사용자와 상호작용이 가능한 상태
    - 특정 이벤트가 발생하여 앱에서 포커스가 떠날 때까지 이 상태에 머무르게 됨
        - 앱 사용중 전화 오거나, 기기 화면 꺼질때
- onPause (휴대폰 작업 탭 일때)
    - 사용자가 Activity를 떠나는것을 나타내는 첫번째 신호
        activity가 포그라운드에 있지 않다는 신호
    - 잠시 작업을 중지해야하는 작업을 일시 중지한다
        - 영상 일시 중지, GPS 일시 중지
- onStop
    - activity가 더 이상 사용자에게 표시가 되지 않는상태
        - 새로운 activity가 나오는 경우
        - 화면이 가려지는 경우
    - 사용자에 보이지 않는 동안 앱은 리소스를 해제 해야한다
        - 영상종료, GPS종료
    - 마지막으로 해야 할 일 수행
        - 저장을 하는 기회, 데이터 처리?
- onDestroy
    - activity가 소멸되기 전에 호출된다
        - 기기회전 or 멀티 윈도우 모드에서 시스템이 일시적으로 activity를 소멸시키는 경우
        - onStop에서 해제되지 않은 모든 리소스를 해제해야한다.
