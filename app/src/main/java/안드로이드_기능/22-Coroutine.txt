코루틴 (co routine)
- 코틀린만 지원하는 개념은 아니다

- 루틴
    - 함수의 진입점과 출구점이 한개인 경우
- 코루틴
    - 함수의 진입점과 출구점이 복수개인 경우, 즉 루틴의 집합

- 장점
    - 비동기 처리에 좋다 (간단하게 작성 가능)
        - 비동기(async) 처리를 순차적으로 처리할 수 있는 문법을 제공
        - 아래와 같은 코드를 가능하게 해준다
            val user = fetchUserData() //네트워크 작업
            textView.text = username //UI 업데이트
            -> 콜백 지옥 해결

- 비동기로 실행을 간단하게 처리할 수 있는 동시 실행 설계 패턴
    -> 루틴으로 묶여있지만 안과 밖을 왔다 갔다 할 수 있다
    - 협력형 멀티 태스킹
    - 동시성 프로그래밍
    - 쉬운 비동기 처리
